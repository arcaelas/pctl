### docs/guides/configuration.md
- Full pctl.yaml anatomy: name, resolver, plugin, custom, services
- Service fields: image, command, env, scale (replica, cpu, memory), ports, health (interval, command, retries, onfailure), volumes, registry, provider

- scale.replica as number vs [min, max]
- custom as reusable store with ${self:custom.*}
- Real complete YAML example with all fields

### docs/guides/resolvers.md
- What resolvers do (process ${name:key, fallback} patterns)
- Built-in: env, ssm, self, cfn
- Each resolver with syntax, examples, fallback behavior
- Nesting: ${env:${ssm:/path}, fallback}
- Resolution order: innermost first
- Custom resolvers: class extends Function, constructor(parsed), return named function
- Adding custom resolvers via resolver array in YAML

### docs/guides/plugins.md
- What plugins do (pipeline functions that receive parsed)
- Built-in pipeline: [resolve, validate, ...user plugins, aws, docker, gcp]
- Writing a custom plugin: export default async function myPlugin(parsed) {}
- Plugin can mutate parsed
- Adding plugins via plugin array in YAML
- Example: logging plugin, env-override plugin

### docs/providers/overview.md
- What a provider does (deploy/destroy services to a target)
- Provider in service: { name, options }
- Built-in: aws, gcp, docker
- How providers detect their services (filter by provider.name)
- State file: pctl.{name}.json
- Fingerprint diff: skip unchanged services
- Labels: managed-by, pctl-stack, pctl-service

### docs/providers/aws.md
- Requirements: AWS CLI or credentials, EKS cluster, ECR or external registry
- Options: credentials (optional), cluster, namespace, strategy, serviceAccount, rbac, storage
- Credentials: from options or environment/AWS config
- What it creates: Namespace, Deployment, Service, HPA, PVC, PV, RBAC (SA+Role+RoleBinding), imagePullSecret
- Storage: EBS (size) vs EFS (name+id) vs ephemeral (no storage)
- ECR auto-auth vs external registry with imagePullSecret
- Health: liveness probe (restart) vs readiness probe (stop)
- Full YAML example with all AWS features
- Destroy behavior: waits for pods, cleans namespace, cleans images

### docs/providers/gcp.md
- Requirements: gcloud CLI authenticated, GKE cluster, Artifact Registry or external registry
- Options: project, zone, cluster, namespace, strategy, serviceAccount, rbac, storage
- Auth via gcloud (no explicit credentials needed)
- Storage: PD (standard-rw) vs Filestore (nfs)
- Artifact Registry auto-auth
- Full YAML example
- Destroy behavior

### docs/providers/docker.md
- Local: provider options empty, docker must be running
- Remote SSH: host, user, key, sudo
- Image transfer: with registry (push/pull) vs without (docker save/scp/load)
- Replica: number creates N containers with suffix -1, -2, etc. Array [min,max] uses max.
- Health: --health-cmd, --health-interval, --health-retries
- onfailure: restart (--restart unless-stopped) vs stop (--restart no)
- sudo: true for servers where docker needs root
- Full YAML example local + remote
- Destroy behavior
